     1→# Earendel — Worklog & Build Plan
     2→
     3→## Product Vision
     4→Earendel is **a reliability layer that turns repeated authorized business workflows
     5→into typed, monitored, repairable tools for AI agents.** Not "an agent that clicks
     6→everywhere" — an atelier that compiles human workflows into typed actions
     7→(inputs/outputs/permissions/tests) backed by a multi-adapter execution engine
     8→(official API → discovered internal route → browser → vision → human review),
     9→with continuous validation, automatic repair, API-style versioning and
    10→risk-gated autonomy. Publishes as MCP tools / REST / SDK.
    11→
    12→## Architecture
    13→- **Frontend** — Next.js 16 (port 3000), single `/` route Studio with internal
    14→  view switching. Design system: dark warm theme (#1F1A17 / #E8E0D4 / #6B5876 /
    15→  #7A8548 / #42403D), Cormorant Garamond (headings) + Hanken Grotesk (body),
    16→  Octicons.
    17→- **Backend** — FastAPI modular monolith (port 8001). CORE = typed-actions
    18→  engine (domain, contracts, engine, validation, repair, versioning, registry).
    19→  SIDE = feature modules (connectors, recordings, actions, executions,
    20→  monitoring, publishing, auth). Adapters = pluggable execution backends.
    21→  SQLite via SQLAlchemy for persistence.
    22→- **Realtime** — socket.io mini-service (port 3003) for live execution streaming.
    23→- **Gateway** — Caddy. Frontend calls backend via `/api/v1/...?XTransformPort=8001`.
